チャットアプリに「明日の予定を整理して、関係者にリマインドを送っておいて」と打つと、AIが実際にカレンダーを読み、メッセージを送る。話すだけで終わらず、手を動かす。OpenClawはそういうエージェントだ。

公開から半年で30万を超えるGitHubスターを集め、AIエージェント界隈で一気に名前が知られた。一方で「自分のマシンにフルアクセス権を持つAIを常駐させる」という性格から、セキュリティ面の議論も同じ熱量で起きている。何ができて、何が危ういのかを整理しておく。

---

## OpenClawとは何か

OpenClawは、自分のデバイス上で動かす個人用AIエージェントだ。PSPDFKitの創業者であるオーストリアの開発者 Peter Steinberger が公開した。MITライセンスのオープンソースで、本体はTypeScript、モバイル連携部分はSwiftで書かれている。

特徴を一言で言えば「ローカルファースト」だ。会話の記憶も設定もMarkdownファイルとして自分のディスクに置かれる。クラウド側のSaaSにデータを預ける構成ではなく、手元のマシンが処理と保存の中心になる。

Claude CodeやGooseと同じ「AIエージェント」というくくりではあるが、射程が違う。コーディング支援に特化したツールではなく、日常的に使うチャットアプリを入口にして、生活や業務まわりの作業全般を任せることを狙っている。Gooseがチーム構造に与える影響については[GooseとAI開発が解体するピラミッド型組織]({% post_url 2026-06-11-goose-agent-pyramid-hierarchy %})で書いたが、OpenClawはそもそも開発チーム向けというより個人向けの色が濃い。

---

## 何ができるか

OpenClawはLLMを「実際の操作」につなぐゲートウェイとして振る舞う。具体的には次のようなことをこなす。

- シェルコマンドの実行、ファイルの読み書き
- ブラウザの操作（フォーム入力やサイトからのデータ抽出）
- メールやカレンダーの操作、予定の調整
- 複数ステップにまたがるワークフローの自律実行

接続先のチャットアプリも幅広い。WhatsApp、Telegram、Slack、Discord、Signal、iMessage、Microsoft Teams、Matrix、LINE、WeChatなど20以上のチャネルに対応し、普段使っているアプリからそのまま話しかけられる。画像・音声・ドキュメントといったメディアも扱える。

自分でSkill（後述）を書き足してできることを増やせるため、機能は固定ではない。必要な操作を新しいSkillとして自身に追加していく作りになっている。

## 仕組み ── ローカルGatewayとSkill

OpenClawの中心にあるのは、手元で動く単一のGatewayプロセスだ。セッション・チャネル接続・ツール呼び出し・イベントを一括で握る「単一の真実の源」として動く。チャットアプリから来たメッセージをエージェントに橋渡しし、複数チャネルを同時にさばく。

インストールはnpm経由で、オンボーディング時にOSのサービス（launchd / systemd）としてデーモン登録される。

```bash
# インストール（Node 24 推奨、22.19+ でも可）
npm install -g openclaw@latest

# オンボーディングとデーモン登録
openclaw onboard --install-daemon
```

設定の最小単位はどのモデルを使うかの指定だけだ。プロバイダのAPIキーを用意し、使うモデルを書く。

```json
{ "agent": { "model": "<provider>/<model-id>" } }
```

ドキュメントは「最新世代の最も強いモデルを使うこと」を推奨している。エージェントの判断力がそのまま安全性に直結するためだ。

エージェントの振る舞いは、ワークスペース直下のMarkdownファイルで定義する。

```
~/.openclaw/workspace/
├── AGENTS.md          # 振る舞いの基本ルール
├── SOUL.md            # 人格・口調
├── TOOLS.md           # 使えるツールの定義
└── skills/
    └── <skill>/SKILL.md
```

Skillは `skills/<skill名>/SKILL.md` という移植可能な形式で書く。記憶も設定もMarkdownで持つので、中身が人間に読めるし、Gitで管理することもできる。Control UIは `http://127.0.0.1:18789/` でローカルに立ち上がり、リモートからアクセスしたい場合は[Tailscale]({% post_url 2026-06-10-tailscale-guide %})越しにつなぐ構成が案内されている。

セッションはエージェント・ワークスペース・送信者ごとに分離できる。グループチャットで誰のメッセージにどのエージェントを割り当てるか、といったルーティングも組める。

## 名前が二転三転した経緯

OpenClawは名前の変遷が激しいプロジェクトとしても知られている。

もとは2025年11月に別名で公開され、Steinbergerの以前のプロジェクト「Clawd」の系譜にある。このClawdという名前がAnthropicのClaudeに着想を得たもので、ここが後で問題になった。何度かのリネームを経て、2026年1月にAnthropicからの商標上の指摘を受けてMoltbotに改名。その数日後、語感が気に入らないという理由で現在のOpenClawに落ち着いた。

2026年2月にはSteinberger自身がOpenAIに加わることを表明し、プロジェクトはコミュニティガバナンスの非営利財団へ移行した。作者が離れてもOSSとして続く体制になっている。

名前の変遷は単なる小話ではなく、「ClaudeにインスパイアされたツールがClaudeとは別の陣営の手で広まり、最終的に作者がOpenAIへ移った」という、いまのAIエコシステムの力学がそのまま透けて見える経緯でもある。

## フルアクセスの代償

OpenClawの便利さは、そのまま危うさの裏返しになっている。デフォルトではメインのセッションにフルアクセス権が与えられる。シェルを叩き、ファイルを書き換え、Skillを連鎖させる ── これを人間が一手ごとに確認せずに進められるのが売りであり、同時にリスクでもある。

Malwarebytesなどが指摘している具体的な懸念は次のあたりだ。

- **過大な権限**: ブラウズ・コマンド実行・ファイル操作を人間のチェックなしで連鎖させられる
- **認証情報の窃取**: インフォスティーラー型マルウェアがAIエージェントの設定を直接狙い始めている。パスワードだけでなくエージェントの人格や鍵ごと盗み、なりすます事例が報告されている
- **プロンプトインジェクション / ログ汚染**: エージェントが処理するメール・サイト・ログに仕込まれた指示で、振る舞いを乗っ取られうる
- **暴走**: あるAIセーフティ研究者は、エージェントが許可なくメール受信箱の大部分を削除したと報告している

このうちプロンプトインジェクションや暴走は、エージェントに自律性を持たせる以上どのツールにも共通する構造的な問題だ。報酬ハッキングやアライメントの話とも地続きで、[AIエージェントの報酬ハッキングとアライメント]({% post_url 2026-06-08-agent-reward-hacking-alignment %})で扱ったテーマがそのまま実運用の足元に現れている。

対処の基本は権限の絞り込みに尽きる。OpenClaw自身も非メインのセッションをサンドボックス内で動かす設定を持っており、Dockerをデフォルトのバックエンドにできる。

```text
# 非メインセッションをサンドボックス化する設定の例
agents.defaults.sandbox.mode: "non-main"
```

加えて、VM内での隔離・ネットワークのallowlist化・本番データへのアクセスを持たないサービスID・Skillの導入元の検証・ログの定期確認といった運用が推奨されている。オランダのデータ保護当局は、機微なデータを扱うシステムへの導入は避けるよう警告まで出している。要するに、便利だからといってメインマシンに無防備に常駐させるツールではない。

## コーディングエージェントとの違い

同じ「AIエージェント」でも、OpenClawとコーディング特化のツールでは狙う場所が違う。

| ツール | 主な入口 | 射程 | LLM | ライセンス |
| :--- | :--- | :--- | :--- | :--- |
| OpenClaw | チャットアプリ | 生活・業務全般の自動化 | 任意（設定） | OSS（MIT） |
| Claude Code | ターミナル・IDE | コーディング特化 | Claude | 商用 |
| Goose | CLI | コーディング＋自動化 | 任意（設定） | OSS（Apache 2.0） |

Claude Codeはコードを書く現場に深く入り込むツールで、エディタやターミナルが入口になる。OpenClawはそこではなく、普段のチャットアプリを入口にして「コード以外も含めた身のまわりの作業」を任せる方向に振れている。GooseはCLIベースでコーディングと自動化の両方をカバーし、立ち位置としてはこの二つの中間に近い。

OpenClawが向くのは、メッセージを送るだけで定型作業を片付けたい個人ユースだ。逆に、扱うデータが機微だったり、組織のマシンに入れたりする場合は、サンドボックス前提で慎重に構える必要がある。

---

## まとめ

OpenClawは「話すだけで実際に手を動かすAIを、自分のマシンにローカルで持つ」という体験を、オープンソースで手の届く形にした。Markdownベースの記憶とSkill、20以上のチャットアプリ連携、財団によるコミュニティ運営 ── ここまで揃ったツールが半年で30万スターを超えたのは、その需要の大きさの裏返しだ。

ただし、便利さとフルアクセスの危うさは同じコインの裏表だ。試すなら、まずサンドボックスと権限の最小化から入るのが現実的なところになる。
